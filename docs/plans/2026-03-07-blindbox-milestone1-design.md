# Blindbox Milestone 1 Design

**Topic:** 清理未来脏数据 + 禁止未来日期运行 + 新增 10/10/20 主策略

**Date:** 2026-03-07

## Goal

把当前 blindbox 从“可跑通”提升到“可信实验台”：

- 清理我测试时写入的未来日期脏数据
- 禁止未来日期再次写入正式状态
- 保留随机策略做对照组
- 新增一个更像样的基准策略：`10%止盈 / 10%止损 / 20天强平`

## Constraints

- 继续只操作 `paper_portfolio`
- 不碰实盘、不碰真实券商
- 保持现有 blindbox 独立日志链路
- 尽量复用现有 `blindbox_*` 模块，不另起一套系统

## Options

### Option A: 只清理数据，不改策略

优点：
- 改动小
- 风险低

缺点：
- 只能解决“脏数据”问题
- 无法把 blindbox 推进到更有价值的实验状态

### Option B: 清理数据 + 禁止未来日期 + 新增基准主策略

优点：
- 同时解决数据可信性和实验价值
- 保留随机策略做对照，便于后续比较
- 最符合当前目标：先把 blindbox 做成可信实验平台

缺点：
- 比单纯清理数据多一点改动
- 需要把出场规则从“固定持有天数”扩展到“止盈 / 止损 / 强平”

### Option C: 直接切到复杂 AI 驱动策略

优点：
- 看起来更接近最终目标

缺点：
- 现在样本、保护和评估都不够稳
- 会让 blindbox 再次变成“看起来聪明，但不可信”

## Recommendation

选择 **Option B**。

原因：
- 先保证 blindbox 的时间线和状态可信
- 再把一个真正可解释的规则策略接进赛马池
- 这样后面比较“随机策略 vs 规则策略 vs AI 策略”时才有意义

## Design

### 1. 数据清理

新增一个 blindbox 维护工具，对以下文件做按“参考交易日”清理：

- `data/blindbox_positions.json`
- `data/blindbox_daily_report.jsonl`
- `data/blindbox_runner_history.jsonl`
- `data/blindbox_runner_latest.json`

规则：
- 删除所有 `trade_date / buy_date / planned_exit_date / last_trade_date` 晚于参考交易日的记录
- 保留不晚于参考交易日的历史

### 2. 未来日期保护

在 `blindbox_daily_runner.py` 中加入保护：

- 显式传入 `--date` 时，若晚于参考交易日，直接拒绝运行
- 正常 `--once` 模式只能使用“最近交易日”

### 3. 新增主策略

新增一个基准策略：

- 内部 ID：`tp10_sl10_t20`
- 中文显示：`10%止盈 / 10%止损 / 20天强平`

执行逻辑：
- T 日收盘后形成信号
- T+1 开盘模拟买入
- 后续每日检查：
  - 高点达到 `+10%`：止盈
  - 低点达到 `-10%`：止损
  - 若同日同时命中：**先止损**
  - 持有达到 `20` 个交易日：强平

### 4. 保留随机策略

保留现有随机策略作为对照组：

- `抛硬币买入`
- `随机买入持有2天`
- 其他现有规则策略

目的不是让随机策略盈利，而是用来确认：
- 管道是否稳定
- 主策略是否真的优于随机对照

### 5. UI 表现

不改底层英文主键，只改显示层：

- `tp10_sl10_t20` 在 UI 中显示中文名
- 清理 future-dated 数据后，blindbox 页面只显示真实时间线记录

## Validation

需要用 3 层验证：

1. **编译级**：模块能成功导入与编译
2. **单测级**：新增规则、清理、未来日期保护测试都通过
3. **实跑级**：blindbox 真跑后只写入不晚于参考交易日的状态
